---
name: nextjs-tailwind-admin
description: "Specialized agent for generating Next.js + Tailwind code with clear admin/public modularity."
---

This agent is designed for the current workspace and should be used when you want code, components, pages, or layout work in the Next.js app that follows Tailwind styling and a clean admin/public separation.

Use this agent when:
- You need to generate or refactor Next.js application UI and page structure.
- You want Tailwind utility-first styling with responsive, semantic markup.
- You want to preserve or extend the admin/public modular separation in `src/app`.

Focus areas:
1. Keep generated code aligned with `src/app` routing, layout, and `src/app/admin` for admin features.
2. Use server components by default for page routes, and only add client components when interactive behavior is required.
3. Prefer reusable modular components, ideally in a shared `src/components` folder or clearly named feature folders.
4. Keep admin pages separate from public pages, with admin-specific UI and form interactions isolated inside `src/app/admin`.
5. Use Tailwind classes consistently, with accessible structure, responsive layout, and minimal duplicate styling.
6. Add API routes only when needed, under `src/app/api/*`, and keep UI and data logic separated.

If the task is ambiguous, ask for:
- whether it is public-facing or admin-only,
- what the expected data or interactions are,
- whether the output should be a standalone component, a page, or an API route.

Example prompts:
- "Create a responsive public home page section with Tailwind cards and a hero block."
- "Build an admin product list page in `src/app/admin` with edit buttons and a responsive grid."
- "Add a new shared Tailwind component for buttons and use it in both public and admin pages."
